# 每周文件汇总说明

## 📁 文件夹结构

```
项目根目录/
├── Week1/          # 第1周：基础框架与用户系统
├── Week2/          # 第2周：报销申请功能
├── Week3/          # 第3周：审核与导出功能
├── Week3.5/        # 第3.5周：完善与本地测试
├── Week4/          # 第4周：云端部署准备
└── Week5/          # 第5周：生产环境优化
```

## 📋 使用说明

### 1. 每周文件夹内容

每个周文件夹包含：
- `第X周工作计划.md` - 详细的工作计划和任务安排
- `文件清单.md` - 该周需要创建/更新的文件清单

### 2. 如何按照计划开发

#### 第1周
1. 阅读 `Week1/第1周工作计划.md`
2. 查看 `Week1/文件清单.md` 了解需要创建的文件
3. 参考项目根目录下的实际文件进行开发
4. 按照每日任务逐步完成

#### 第2-5周
同样的流程，每周按照对应文件夹中的计划进行。

### 3. 文件参考

**重要**：每周需要创建的文件，其具体内容可以参考项目根目录下的对应文件。例如：
- `Week1` 的文件可以参考根目录下的 `users/` 文件夹
- `Week2` 的文件可以参考根目录下的 `claims/` 文件夹
- `Week4-5` 的部署文件可以参考根目录下的 `deploy.sh` 等文件

### 4. 迭代开发建议

1. **每周开始前**：
   - 阅读本周工作计划
   - 了解本周目标和任务
   - 准备开发环境

2. **开发过程中**：
   - 按照每日任务逐步完成
   - 及时测试功能
   - 记录遇到的问题

3. **每周结束时**：
   - 完成本周验收标准
   - 修复已知Bug
   - 准备下周工作

### 5. 关键里程碑

- **第1周末**：用户系统可用
- **第2周末**：报销申请功能完成
- **第3周末**：审核导出功能完成
- **第3.5周末**：本地版本完成
- **第4周末**：成功部署到云端
- **第5周末**：生产环境优化完成，项目交付

## ⚠️ 注意事项

1. **不要跳过步骤**：每周的工作是循序渐进的，建议按顺序完成

2. **及时测试**：每完成一个功能就进行测试，不要等到最后

3. **记录问题**：遇到问题及时记录，便于后续解决

4. **代码备份**：每周结束前备份代码，防止丢失

5. **灵活调整**：如果遇到技术难点，可以适当调整计划，但要保证总体进度

## 📞 需要帮助？

如果遇到问题：
1. 查看对应周的文件清单和计划
2. 参考项目根目录下的实际代码
3. 查阅Django官方文档
4. 搜索相关技术问题

祝开发顺利！🎉
